# Error E107: Cycle detected in flat post-expansion graph

## What it means

After expanding all composition call sites and flattening the graph, a
cycle was detected. This refines E003 (pre-expansion cycle detection)
with hierarchical name context showing which composition bodies
contribute to the cycle.

## Example

A composition that (directly or indirectly) references itself:

```yaml
# loop.comp.yaml
_compose:
  inputs:
    data: { schema: [{ name: id, type: string }] }
  outputs:
    data: { schema: [{ name: id, type: string }] }
nodes:
  - type: composition
    name: recurse
    input: data
    use: ./loop.comp.yaml  # self-reference -> cycle
```

## How to fix

Break the cycle by removing the self-referential composition call or
restructuring the pipeline to avoid circular dependencies.

## Technical context

Per LD-16c-6, cycle detection runs on the flat post-expansion graph, not
the hierarchical representation. `PipelineNode::Composition` boundary
markers are retained as structural no-ops but elided during cycle
analysis. This avoids the false positives and false negatives that arise
from scope-qualified name resolution on hierarchical graphs (a known
issue in Airflow TaskGroup).
