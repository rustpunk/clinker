# Error E306: Combine drive hint references unknown input

## What it means

The `drive:` field on a combine node names a qualifier that is not
present in the `input:` map. The planner uses `drive:` to override
its automatic driving-input selection; if the named qualifier doesn't
exist, the hint is meaningless.

## Example

```yaml
# Invalid: drive names a qualifier that is not in input:
nodes:
  - type: combine
    name: bad
    input:
      orders: orders
      products: products
    where: "orders.product_id == products.id"
    drive: "customers"                 # ERROR: no "customers" input
    cxl: |
      emit orders.id, products.name
```

```yaml
# Corrected: drive names one of the declared qualifiers
nodes:
  - type: combine
    name: good
    input:
      orders: orders
      products: products
    where: "orders.product_id == products.id"
    drive: "products"                  # products is the smaller side
    cxl: |
      emit orders.id, products.name
```

## How to fix

Either remove the `drive:` hint (let the planner pick automatically)
or set it to one of the qualifier names declared in the `input:` map.
The driving input is the one whose rows drive the join loop; for
HashBuildProbe this is the probe side. Pick the smaller input when you
have cardinality information the planner lacks.

## Technical context

Driving-input selection is cost-based: the planner picks the input
with the smallest estimated cardinality as the build side (for
HashBuildProbe) or the outer loop (for nested-loop strategies). When
cardinality estimates are tied or absent, the planner emits W306 (not
an error) and picks arbitrarily. `drive:` lets the user override this.
E306 validates the hint at bind_schema time by membership-checking
against the `input:` qualifier set; the check happens before strategy
selection so an invalid hint never reaches the planner.
