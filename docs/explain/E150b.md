# Error E150b: windowed transform references field outside upstream's anchor schema

## What it means

A windowed Transform whose first non-pass-through ancestor is an
operator other than a Source (`Aggregate`, `Combine`, `Transform`,
`Composition`) declares an `analytic_window:` whose `partition_by`,
`sort_by`, or referenced fields name a column the upstream operator
does not emit. The windowed Transform's arena is materialized from the
upstream's emit buffer; a column that is not in the upstream's
`output_schema` would silently resolve to `Value::Null` at runtime.

## Example

```yaml
nodes:
  - type: source
    name: src
    config:
      schema:
        - { name: department, type: string }
        - { name: amount, type: int }
  - type: aggregate
    name: dept_totals
    input: src
    config:
      group_by: [department]
      cxl: |
        emit department = department
        emit total = sum(amount)
  - type: transform
    name: bad_window
    input: dept_totals
    config:
      analytic_window:
        group_by: [region]    # E150b — `region` is not on dept_totals' output schema
      cxl: |
        emit running_total = $window.sum(total)
```

## How to fix

Either remove the offending field from `partition_by` / `sort_by` /
window-referenced expressions, or extend the upstream operator to emit
the column. For an aggregate, add the column to its `group_by` or its
`emit` body.

## Technical context

Pre-rip the analytic-window arena was always rooted at the source; a
field reference downstream of an aggregate would silently resolve to
Null because the source schema did not carry the aggregate-emitted
column. Post-rip the arena roots at the upstream operator
(`PlanIndexRoot::Node` / `ParentNode`), and the lowering pass at
`config/mod.rs` validates `arena_fields ⊆ anchor_schema`. The same
validation runs inside composition bodies via
`resolve_composition_body_windows` so body windows fail fast on the
same shape.
