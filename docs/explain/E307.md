# Error E307: Combine input references undeclared upstream

## What it means

One of the values in a combine node's `input:` map references a
node name that does not exist anywhere in the pipeline. The combine
node's DAG edges cannot be wired because there is no upstream
producer to connect to.

## Example

```yaml
# Invalid: "prdoucts" — the source is named "products"
nodes:
  - type: source
    name: orders
    config: { type: csv, path: orders.csv, schema: [...] }
  - type: source
    name: products
    config: { type: csv, path: products.csv, schema: [...] }

  - type: combine
    name: enrich
    input:
      orders: orders
      products: prdoucts           # ERROR: undeclared (typo)
    where: "orders.product_id == products.id"
    cxl: |
      emit orders.id, products.name
```

```yaml
# Corrected: match the actual source name
nodes:
  - type: combine
    name: enrich
    input:
      orders: orders
      products: products           # now resolves
    where: "orders.product_id == products.id"
    cxl: |
      emit orders.id, products.name
```

## How to fix

Check the spelling of the upstream reference against the `name:`
fields of the pipeline's source, transform, aggregate, route, merge,
composition, and combine nodes. Remember that combine inputs can
reference any producer node type — not just sources. Common causes:
- Typo in the node name.
- Referenced node was removed or renamed elsewhere in the file.
- Referenced node lives in a composition that wasn't imported.

## Technical context

E307 fires during DAG-edge wiring in `ExecutionPlanDag::compile()`
when `resolve_any_node()` cannot match the input value against any
declared producer. Unlike transform's single-input `input:` field
(which uses `resolve_input_reference` — transform-like prefixes
only), combine's multi-input map accepts any node type as a producer,
so a broader resolver is needed (RESOLUTION W-2).

The diagnostic's primary span points at the offending value in the
`input:` map; secondary spans list the closest name matches in the
pipeline's node list.
