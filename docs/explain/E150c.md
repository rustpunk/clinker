# Error E150c: cross-source window references a source whose ingestion tier is downstream

## What it means

A cross-source `analytic_window: { source: <other> }` declares that the
window evaluates against rows of a different source. The referenced
source MUST be ingested before the windowed Transform's primary input
source — otherwise the engine would attempt to project against an
arena that has not yet been built. The diagnostic fires when the
ingestion-tier ordering is inverted.

## Example

A cross-source window where the target source is in a later
`build_source_dag` tier than the windowed transform's primary source.

```yaml
- type: transform
  name: enriched
  input: orders
  config:
    analytic_window:
      source: late_arrival     # E150c if `late_arrival` is in a later tier than `orders`
      group_by: [department]
      on: department
```

## How to fix

Either:

* Promote the referenced source to an earlier tier (today,
  `build_source_dag` puts every cross-source-referenced source in tier
  0 automatically — the diagnostic acts as defense-in-depth against
  future tier-inversion geometries).
* Remove the cross-source reference and root the window at the primary
  input.

## Technical context

`build_source_dag` orders sources into tiers so cross-source-referenced
sources are ingested before consumers. Today's implementation always
yields a two-tier graph with referenced sources in tier 0 and primary
sources in tier 1, so user-authored YAML cannot construct a
tier-inverted geometry. E150c is wired as a guard against extensions
that admit multi-tier inversions; the unit test at
`crates/clinker-core/src/executor/tests/cross_source_window_topology.rs`
pins the predicate's behavior on a synthetically-constructed multi-tier
input.
