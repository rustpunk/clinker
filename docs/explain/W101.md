# Warning W101: Pass-through column shadowed by composition body column

## What it means

An upstream row carries a column that passes through a composition's input
port, but the composition body creates a column with the same name. The
body column wins (shadows the pass-through), and this warning is emitted
so the author is aware of the collision.

## Example

```yaml
# Upstream row has columns: id, name, tag
# Composition body adds: tag (via add_fields)
# Result: body's "tag" shadows upstream's "tag" -> W101
```

## How to fix

Either rename the body-created column to avoid the collision, or accept
the shadowing (the warning is informational). If the shadowing is
intentional, no action is needed -- the warning confirms the body's value
takes precedence.

## Technical context

Per LD-16c-2 (port schema: minimum-required + body-wins), upstream rows
may carry additional columns beyond the declared port schema. These
extras pass through the composition boundary. When the body creates a
column with the same name as a pass-through column, the body column wins.
This warning (W101) is informational — it confirms the shadowing rather
than blocking compilation, since the body-wins rule is by design.
