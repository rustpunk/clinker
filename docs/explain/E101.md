# Error E101: Composition signature parse error

## What it means

The `.comp.yaml` file defining a composition could not be parsed. This
may be due to malformed YAML, missing required fields (`_compose:`,
`inputs:`, `outputs:`), or a type mismatch in the declared port schema.

## Example

```yaml
# Missing required _compose: header
nodes:
  - type: transform
    name: t1
    input: data
    config:
      add_fields:
        tag: "x"
```

## How to fix

Ensure the `.comp.yaml` file has a valid `_compose:` header with
`inputs:` and `outputs:` sections, and that all declared schema fields
use valid types.

## Technical context

Composition signatures are parsed during Phase 1 of the two-phase compile
process — a workspace-wide scan that builds the symbol table. The
`_compose:` header declares `inputs:`, `outputs:`, `config_schema:`, and
`resources_schema:`; config and resources are two separate axes, so they
occupy two distinct slots rather than a single merged map. A parse failure
here blocks the entire Phase 1 symbol table build because downstream call
sites and channels cannot resolve against a malformed signature.
