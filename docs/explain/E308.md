# Error E308: Combine cxl body references unknown field

## What it means

The `cxl:` body of a combine node references a qualified field
`<qualifier>.<field>` that is not present in the merged row. Either
the referenced field was never produced by any input, the qualifier
is wrong, or the name is misspelled.

## Example

```yaml
# Invalid: cxl body references orders.total, but orders has no "total" field
nodes:
  - type: source
    name: orders
    config:
      type: csv
      path: orders.csv
      schema:
        - { name: id,         type: string }
        - { name: product_id, type: string }
        - { name: qty,        type: int }
  - type: source
    name: products
    config:
      type: csv
      path: products.csv
      schema:
        - { name: id,    type: string }
        - { name: price, type: decimal }

  - type: combine
    name: bad
    input:
      orders: orders
      products: products
    where: "orders.product_id == products.id"
    cxl: |
      emit orders.id, orders.total       # ERROR: orders.total not in merged row
```

```yaml
# Corrected: compute total from available fields
nodes:
  - type: combine
    name: good
    input:
      orders: orders
      products: products
    where: "orders.product_id == products.id"
    cxl: |
      let total = orders.qty * products.price
      emit orders.id, total
```

## How to fix

Check each `<qualifier>.<field>` reference in the `cxl:` body against
the schemas of the inputs. If the field should exist but does not,
inspect the upstream node's schema in Kiln or run `clinker guess`.
Derived fields must be computed in the body (`let … = …; emit …`),
not assumed into existence.

## Technical context

E308 is the `cxl:` body counterpart of E304 (which fires on `where:`
references). Both check the same merged-row schema, but at different
phases: E304 runs during where-clause typecheck, E308 runs during
body typecheck, which happens after the where-clause has been
decomposed and classified. Splitting the two codes lets users
pinpoint whether to look at `where:` or `cxl:` first.
