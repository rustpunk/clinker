# Error E104: Call site or channel missing required input/config

## What it means

A composition call site or channel binding does not provide a value for a
required parameter (one without a default) declared in the target
composition's signature.

## Example

```yaml
# customer_enrich requires lookup_table (no default)
- type: composition
  name: enrich
  input: src
  use: ./customer_enrich.comp.yaml
  config:
    fuzzy_threshold: 0.9
    # missing: lookup_table
```

## How to fix

Add the missing required parameter to the call site's `config:`,
`inputs:`, or `resources:` map.

## Technical context

Per LD-16c-1, composition parameters are declared in `config_schema:` with
`required: true|false` and optional `default:` values. A parameter marked
`required: true` with no default must be supplied at every call site and
cannot be deferred to channel overlays. This ensures that a composition
compiles standalone without relying on external configuration.
