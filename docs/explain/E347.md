# Error E347: reconstruct_envelope is incompatible with this pipeline shape

## What it means

An Output declares `reconstruct_envelope: true` while the pipeline also uses a
feature that owns the Output's writes in a way envelope reconstruction cannot
honor. Envelope reconstruction frames one document stream into a **single**
writer — emitting a per-document header before each document's first record and
a footer after its last — so it is incompatible with anything that routes the
same records to a different writer or buffers them under a different commit
model. Three combinations are rejected:

- **Per-file output `split:` / a per-source-file path template**
  (`{source_file}` / `{source_path}`). These fan each record out to a writer
  keyed by its file. The envelope arm only ever consults the single-writer
  registry, so under fan-out it finds no writer and silently drops every record
  while the run counters still report them written — total silent data loss.

- **A source declaring `dlq_granularity: document`.** Document-level
  dead-lettering routes the Output through its own per-document buffer, which
  takes precedence over envelope framing, so the header/footer hooks never fire
  and reconstruction silently does nothing.

- **A source declaring `correlation_key`.** Correlation buffering owns the
  writes through its own commit stage, which the envelope arm bypasses. Dirty
  correlation groups would leak into the framed output and inflate the success
  counts.

```
[E347] output 'out': `reconstruct_envelope` cannot be combined with per-file
output splitting (`split:`) or a per-source-file path template (`{source_file}`
/ `{source_path}`) — envelope reconstruction frames one document stream into a
single writer, which is incompatible with routing each record to a file-keyed
writer. Use a single output path without `split:`, or drop `reconstruct_envelope`
```

## Example

```yaml
# Rejected: envelope reconstruction with per-file splitting.
pipeline:
  name: claims
nodes:
  - type: source
    name: claims
    config:
      name: claims
      type: x12
      glob: ./claims/*.edi
  - type: output
    name: out
    input: claims
    config:
      name: out
      type: x12
      path: out.edi
      reconstruct_envelope: true
      split:                        # per-file fan-out — incompatible
        max_records: 1000
```

```yaml
# Corrected: a single output path, no split — one framed document stream.
pipeline:
  name: claims
nodes:
  - type: source
    name: claims
    config:
      name: claims
      type: x12
      glob: ./claims/*.edi
  - type: output
    name: out
    input: claims
    config:
      name: out
      type: x12
      path: out.edi
      reconstruct_envelope: true
```

## How to fix

1. **Drop `split:` and any `{source_file}` / `{source_path}` token** from the
   Output path so envelope reconstruction writes one framed stream to a single
   file. (If you genuinely need per-file output, drop `reconstruct_envelope`
   instead.)

2. **Set `dlq_granularity: record`** (the default) on every source feeding an
   envelope Output — document-level dead-lettering and envelope framing both
   want to own the per-document write boundary and cannot coexist.

3. **Drop `correlation_key`** from sources feeding an envelope Output, or drop
   `reconstruct_envelope` — correlation buffering and envelope framing are
   different write-ownership models.

## Technical context

Envelope reconstruction is handled by a dedicated Output dispatch arm that
detects document boundaries from each record's `doc_ctx().source_file()` and
fires the writer's `begin_document` / `end_document` around each document's
records, streaming the body straight through one writer. That single-writer,
record-driven model is structurally incompatible with the per-file fan-out
registry (a different writer per file), the document-DLQ Output buffer (which
short-circuits the Output arm to decide each document clean-vs-rejected before
writing), and correlation buffering (which commits whole groups through its own
terminal). Rather than let any of these silently win — dropping records,
skipping framing, or leaking dirty groups — the validator rejects the pairing at
config-validation time.
