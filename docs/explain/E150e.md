# Error E150e: window builtin references an array-typed field from a `match: collect` combine

## What it means

A windowed Transform downstream of a `match: collect` Combine
references one of the combine's output columns whose CXL type is
`Array`. Window builtins (`$window.sum`, `$window.avg`, `$window.min`,
`$window.max`, `$window.lag`, `$window.lead`, ...) do not handle
`Value::Array`; they would silently see `Value::Null` at runtime.

## Example

```yaml
- type: combine
  name: collected
  input:
    orders: orders
    products: products
  config:
    where: "orders.product_id == products.product_id"
    match: collect
    cxl: ""
    propagate_ck: driver
- type: transform
  name: bad
  input: collected
  config:
    analytic_window:
      group_by: [products]      # E150e — `products` is Array-typed
    cxl: |
      emit any_field = products
```

## How to fix

Either flatten the array upstream — replace `match: collect` with
`match: first` or `match: all` so the build qualifier emits scalar
columns instead of an array — or avoid referencing the array column
from the window. Aggregate-style folds over an array column should
happen inside the body emit, not via `$window.*`.

## Technical context

`match: collect` synthesizes the combine's output row as
`{ driver fields, <build_qualifier>: Array }`. Lowering checks each
windowed Transform's `accessed_fields` against the combine's typed
`output_row` for `Type::Array` and rejects.
