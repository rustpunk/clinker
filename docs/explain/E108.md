# Error E108: Composition body references enclosing scope (IsolatedFromAbove)

## What it means

A node inside a composition body references a name from the enclosing
pipeline's scope. Composition bodies are isolated from above -- they can
only reference names declared in their own `_compose.inputs:` section and
nodes defined within the body.

## Example

```yaml
# In a pipeline with a source named "global_src"
- type: composition
  name: enrich
  input: global_src
  use: ./my_comp.comp.yaml

# Inside my_comp.comp.yaml:
nodes:
  - type: transform
    name: t1
    input: global_src  # ERROR: referencing enclosing pipeline scope
    config:
      add_fields:
        tag: "x"
```

## How to fix

Use only declared input port names inside the composition body. Pass
external data through the composition's `inputs:` ports at the call site.

## Technical context

The IsolatedFromAbove sealing contract (modeled after MLIR's region
isolation) ensures compositions are self-contained and reusable across
pipelines. A composition body may only reference names from its own
`_compose.inputs:` declarations and nodes defined within the body. This
guarantee makes compositions safe to extract, inline, or rearrange
without ambient scope leaking through.
