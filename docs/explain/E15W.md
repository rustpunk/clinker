# Error E15W: non-deterministic CXL builtin downstream of a retraction-mode aggregate

## What it means

A Transform downstream of an aggregate that activates the retraction
protocol calls a non-deterministic CXL builtin (today: `now`). An
aggregate activates the retraction protocol whenever its `group_by`
omits at least one pipeline-level correlation-key field; the engine
selects this path automatically with no opt-in flag. The retraction
correlation-commit protocol replays the deterministic portion of the
post-aggregate sub-DAG when a triggered group's records get retracted;
a Transform whose output depends on wall-clock time would produce a
different row on replay than at the original execution, breaking the
post-retract substitution proof that the protocol relies on.

## Example

```yaml
error_handling:
  correlation_key: order_id

nodes:
  - type: aggregate
    name: dept_totals
    input: validate
    config:
      group_by: [department]            # omits order_id, so retraction is active
      cxl: |
        emit total = sum(amount)

  - type: transform
    name: stamp
    input: dept_totals
    config:
      cxl: |
        emit emitted_at = now    # E15W — non-deterministic on replay
```

## How to fix

Either move the timestamp stamping upstream of the aggregate (so it
runs once and is stable through commit) or include every
correlation-key field in the aggregate's `group_by` so the aggregate
stays on the strict-collateral path where replay is not needed.

## Technical context

The five-phase correlation-commit protocol detects affected aggregate
groups, retracts the failing rows, refinalizes the aggregator in place,
and replays downstream operators with the new aggregate output rows
substituted for the old ones. Replay only works when every operator on
the path is deterministic — same input, same output. `now` reads
wall-clock time and would produce a different output the second time.
The wider class of non-deterministic builtins (random number
generation, UDFs flagged non-deterministic) gets caught by the same
walker once those features land in CXL.
