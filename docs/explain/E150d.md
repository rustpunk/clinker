# Error E150d: windowed transform rooted at a Merge node

## What it means

A windowed Transform's first non-pass-through ancestor is a
`Merge` node. `Merge` concatenates rows from multiple producers
without preserving a single source identity, so a window cannot anchor
to it — record positions across the merged stream are not well-defined,
and partition assignment by `partition_by` would still work but
position-keyed builtins (`lag`, `lead`, `first`, `last`) would
operate over an arbitrarily-interleaved buffer.

## Example

```yaml
nodes:
  - type: merge
    name: combined
    inputs: [src_a, src_b]
  - type: transform
    name: windowed
    input: combined
    config:
      analytic_window:
        group_by: [department]    # E150d — Merge has no single producer identity
      cxl: |
        emit running_total = $window.sum(amount)
```

## How to fix

Place the windowed Transform upstream of the Merge (one window per
producer, then merge), or use an `Aggregate` to fold the merged stream
before windowing.

## Technical context

Source arenas root at a single producer. Aggregate / Combine / Transform
output buffers do too. Merge violates that invariant — it is a fan-in
without re-keying. The lowering pass at `config/mod.rs` (and its body
counterpart in `resolve_composition_body_windows`) detects Merge as
the rooted ancestor and rejects.
