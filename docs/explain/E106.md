# Error E106: Name collision after composition expansion

## What it means

After expanding (flattening) all composition call sites, two nodes in the
resulting flat graph have the same fully-qualified name. This typically
happens when a composition is called multiple times without distinct
aliases.

## Example

```yaml
- type: composition
  name: enrich_a
  input: src
  use: ./customer_enrich.comp.yaml

- type: composition
  name: enrich_b
  input: src
  use: ./customer_enrich.comp.yaml
  # Both expand internal node "enrich_join" -> collision
```

## How to fix

Add an `alias:` to one or both call sites to give them distinct namespace
prefixes. For example: `alias: a` produces `a.enrich_join` instead of
`enrich_a.enrich_join`.

## Technical context

During Phase 2 body resolution, composition body nodes are renamed with a
namespace prefix derived from the call-site alias (e.g., `enrich_a.join`).
If two call sites produce the same prefix (because they share an alias or
no alias is specified), their internal nodes collide. The `alias:` field
on `PipelineNode::Composition` controls this prefix and must be unique
across sibling call sites using the same composition.
