# Error E309: Combine output schema is empty

## What it means

The `cxl:` body of a combine node produces no output columns — either
it contains no `emit` statements, or every `emit` path is gated by a
condition that never materializes in the output schema. A combine
node must produce at least one column; otherwise downstream consumers
have nothing to consume.

## Example

```yaml
# Invalid: body has a let binding but never emits
nodes:
  - type: combine
    name: bad
    input:
      orders: orders
      products: products
    where: "orders.product_id == products.id"
    cxl: |
      let total = orders.qty * products.price    # ERROR: no emit statement
```

```yaml
# Invalid: emit is unreachable at schema-derivation time
nodes:
  - type: combine
    name: also_bad
    input:
      orders: orders
      products: products
    where: "orders.product_id == products.id"
    cxl: |
      if false then
        emit orders.id
      end                                        # ERROR: constant-false branch
```

```yaml
# Corrected: emit at least one column unconditionally
nodes:
  - type: combine
    name: good
    input:
      orders: orders
      products: products
    where: "orders.product_id == products.id"
    cxl: |
      let total = orders.qty * products.price
      emit orders.id, products.name, total
```

## How to fix

Add at least one unconditional `emit` statement to the `cxl:` body.
If you need conditional shaping, use `if`/`match` around a non-empty
set of emissions rather than gating the entire body.

## Technical context

The output schema of a combine is determined by folding every `emit`
path in the `cxl:` body; each path must contribute the same column
names and types. An empty fold (no reachable `emit`) has no schema to
propagate downstream. E309 fires during CXL typecheck of the body
after schema-shape analysis confirms no path produces output. Per
spec, each combine node must produce a non-empty row — even pass-
through cases must explicitly emit, so authors see and confirm their
output shape.
