# Error E15Y: retraction-mode aggregate incompatible with strategy: streaming

## What it means

An Aggregate node's `group_by` omits at least one pipeline-level
correlation-key field — which routes it through the retraction
protocol so a single correlation group can span multiple aggregate
groups — but its `strategy:` is `streaming`. Streaming aggregates emit
results at every group-boundary close, which happens *before* the
terminal correlation commit. The retraction protocol exists so that
records dropped at `CorrelationCommit` can retroactively retract or
recompute their contributions to surviving aggregate groups; once a
streaming aggregate has flushed a group's row, that rollback window is
closed.

## Example

```yaml
error_handling:
  correlation_key: order_id

nodes:
  - type: aggregate
    name: dept_totals
    input: validate
    config:
      group_by: [department]      # omits order_id, so retraction is active
      strategy: streaming         # E15Y -- incompatible with retraction
      cxl: |
        emit total = sum(amount)
```

## How to fix

Either drop the `strategy: streaming` hint (the `hash` default keeps
results buffered until `CorrelationCommit`, which is what the
retraction path needs), or include every correlation-key field in
`group_by` so the aggregate stays on the strict-collateral path where
streaming is safe.

```yaml
- type: aggregate
  name: dept_totals
  input: validate
  config:
    group_by: [department]      # retraction is active; hash strategy is fine
    cxl: |
      emit total = sum(amount)
```

## Technical context

Today's `CorrelationCommit` flushes per-group buffers at end-of-DAG and
DLQs entire correlation groups when any contributing record fails
upstream. Hash aggregation buffers its accumulator state until
`finalize` and so coexists with the rollback window. Streaming
aggregation emits per group-boundary close, which under retraction
mode would deliver an aggregate row to the writer before the engine
knows whether any of its contributing records will be DLQ'd at commit
time.

The engine selects between the strict-collateral and retraction-mode
paths automatically based on whether `group_by` includes every
correlation-key field; users do not configure this directly. E15Y
fires when the user's chosen `strategy` conflicts with the
auto-selected path.
