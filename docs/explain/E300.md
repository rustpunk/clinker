# Error E300: Combine input count is out of bounds

## What it means

A `combine` node's `input:` map has either fewer than 2 entries or more
than 8. Combine is an N-ary operator over correlated streams: a
single-input "combine" has nothing to match against (use `transform`
for single-stream row operations); a 9+-input combine produces a
decomposition chain whose memory and plan-stability characteristics the
planner does not currently support — split it into two combines
joined through an intermediate node.

## Example

```yaml
# Invalid: only one input
nodes:
  - type: source
    name: orders
    config:
      type: csv
      path: orders.csv
      schema:
        - { name: id, type: string }

  - type: combine
    name: enrich
    input:
      orders: orders      # ERROR: fewer than 2 inputs
    where: "true"
    cxl: |
      emit orders.id
```

```yaml
# Corrected: two inputs with a correlation predicate
nodes:
  - type: source
    name: orders
    config: { type: csv, path: orders.csv, schema: [ ... ] }
  - type: source
    name: products
    config: { type: csv, path: products.csv, schema: [ ... ] }

  - type: combine
    name: enrich
    input:
      orders: orders
      products: products
    where: "orders.product_id == products.id"
    cxl: |
      emit orders.id, products.name
```

## How to fix

Add at least one more input stream under `input:`. If you genuinely only
have one stream, use a `transform` node instead of `combine`.

## Technical context

Combine correlates rows across independent input streams by evaluating a
boolean `where:` predicate over a merged row formed from one row per
input. Per spec §Combine Semantics, the operator is inherently N-ary
with N >= 2 — the planner picks a strategy (HashBuildProbe, IEJoin,
SortMergeJoin, BlockNestedLoop, GraceHash, InMemoryHash) based on the
decomposed predicate shape and input cardinalities, all of which
require at least two inputs to be meaningful.

This check runs during `bind_schema` when lowering `PipelineNode::Combine`
into `PlanNode::Combine`, carrying the `input:` map span for the primary
label.
